# v1.0 Internal Preview User Guide

日期：2026-06-23

适用对象：实验室内部试用者、研究生、项目演示参与者。当前版本是内部预览，
用于走通 approved template review packet、fake lifecycle、workspace/log/artifact
预览和安全边界检查；它不是公网平台，也不是已经接入真实服务器的运行系统。

## What You Can Do

在 v1.0 internal preview 中，用户可以：

- 查看已批准模板的结构化参数边界。
- 填写参数并生成 dry-run review packet。
- 查看 fake lifecycle 状态和下一步允许状态。
- 查看 workspace、run directory、log path 和 artifact path 的 preview。
- 查看 approved-template run packet，确认系统不会执行命令。
- 查看 sanity runner gate 的 review packet，确认未来 fixed runner 需要哪些
  timeout、stdout/stderr capture、artifact path 和 audit metadata。
- 请求删除预览，但只能得到 delete dry-run review packet。

## What You Cannot Do

当前版本禁止：

- 输入自由 shell command 并让系统执行。
- 读取服务器密码、token、私钥、`.env` 或 secret 文件。
- 连接 SSH、Slurm、PBS、远程服务器或真实 single-server account。
- 执行 CUDA/MPI、`mpirun`、`srun` 或 local wrapper。
- 创建 workspace、创建目录、移动文件或采集真实 artifact。
- 删除代码、环境、共享数据、凭据或其他人的实验结果。
- 让 Code Agent 自动应用 patch。

## Choose An Approved Template

当前 approved template 入口是 C++ metadata，不是生产配置库。示例模板在测试中以
`SingleServerJobTemplate` 表达：

- `template_id`: `fwi_multiscale_review`
- `version`: `1`
- `profile_id`: `single-server-dev`
- `entrypoint_label`: `fwi_multiscale_sanity_check`
- allowed parameters: `dataset_id`, `niter`, `frequency_band`, `gpu_count`
- expected artifacts: `loss_curve`, `final_velocity_model`

用户只填写结构化参数。不要写 `mpirun`、`srun`、`ssh`、`rm -rf` 或任何命令字符串。

## Generate A Review Packet

review packet 的核心数据流是：

```text
SingleServerProfile
  + SingleServerJobTemplate
  + SingleServerReviewRequest
  -> render_single_server_review_packet
```

用户应检查 packet 中这些字段：

- `execution: disabled`
- `credentials_loaded: false`
- `server_connection: disabled`
- `workspace_created: false`
- 参数只来自 template allowlist
- credential reference 不会被渲染出来

## Inspect Fake Lifecycle

fake lifecycle 用 `SingleServerLifecycleRecord` 展示状态流：

```text
requested -> reviewed -> approved -> queued -> running -> succeeded
```

也可以走：

```text
requested -> reviewed -> rejected
approved -> cancelled
running -> failed
```

这些状态只是 metadata。`queued` 和 `running` 不代表进入真实队列或真实进程。
用户应检查 preview 中：

- `allowed_next_states`
- `server_connected: false`
- `command_executed: false`
- `workspace_created: false`

## Inspect Workspace Plan

workspace planner 只生成字符串预览：

- planned workspace path
- planned run directory path
- planned log path
- planned artifact paths

必须确认：

- path 在 lab workspace root 下。
- 没有 `..` traversal。
- 没有绝对路径逃逸。
- 没有 `.git`、`.ssh`、`secrets`、`credentials`、`repo`、`code`、`env`、
  `venv`、`shared_data` 等保护标签。
- `directories_created: false`
- `files_moved: false`
- `server_connected: false`

## Inspect Approved Template Run Packet

approved-template run packet 把 profile、template、review request、workspace
plan 和 lifecycle id 合在一起。用户应检查：

- profile/template/version 是否匹配。
- required parameters 是否齐全。
- 未批准参数是否出现在 validation errors 中。
- 自由 command 是否被拒绝且不会渲染。
- `command_executed: false`
- `credentials_loaded: false`
- `server_connected: false`
- `workspace_created: false`
- `directories_created: false`
- `files_moved: false`
- `free_form_command_accepted: false`

## Inspect Sanity Runner Gate

v0.15 只定义 future fixed runner 的 gate，不运行 runner。用户应检查：

- runner id 是否来自 allowlist。
- timeout 是否为正数。
- stdout/stderr capture plan 是否存在。
- artifact path 是否留在 workspace root 下。
- audit event type 是否为 metadata-only review packet。
- 自由命令、删除、凭据读取、SSH、Slurm、PBS、remote server access 是否被拒绝。

## Validate Locally

内部预览的主验证命令：

```bash
cmake --build build -j2
ctest --test-dir build --output-on-failure
```

只看 v1.0 internal preview 相关 gate：

```bash
ctest --test-dir build -R "(SingleServerBackendTest|SafeOperationsTest|SingleServerLifecycleTest|WorkspacePlannerTest|ApprovedTemplateRunPacketTest|InternalSanityRunnerTest)" --output-on-failure
```

通过这些测试只能证明 review-only metadata gate 工作正常，不能证明真实服务器执行可用。

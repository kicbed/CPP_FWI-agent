## v1.0 Internal Preview 定义

这里的 v1.0 是“实验室内部可试用版”，不是公网平台，也不是复杂多用户调度系统。
第一版只服务自己或小组共用一个服务器账号的场景。

进入 v1.0 internal preview 前，系统至少要做到：

- 用户不用写自由 shell command。
- 用户不用接触服务器密码、token、私钥或真实凭据。
- 用户只能选择 approved template，并填写结构化参数。
- 系统生成 review packet，说明将要做什么、不会做什么、有哪些风险。
- 系统有 lifecycle 状态，让用户能看到 requested、reviewed、approved、
  queued、running、succeeded、failed 等状态变化。
- 系统不会删除代码、环境、共享数据、凭据或其他人的实验结果。
- 删除类请求第一版只生成 dry-run review packet，不执行真实删除。
- 真实执行能力最多先进入固定 sanity-check runner，不接受用户自由命令。

## 分步路线

| 顺序 | 版本 | 阶段 | 目标 | 是否执行真实命令 |
| --- | --- | --- | --- | --- |
| 1 | v0.11 | Safe Operations Policy | 实现内部角色、操作 allowlist 和删除 dry-run review packet | 否 |
| 2 | v0.12 | Fake Lifecycle | 实现单服务器 fake lifecycle 状态机和 review 流程 | 否 |
| 3 | v0.13 | Workspace Planner | 生成 workspace/artifact plan 并做路径安全校验 | 否 |
| 4 | v0.14 | Approved Template Run Packet | 把 approved template 和结构化参数渲染为 run packet | 否 |
| 5 | v0.15 | Internal Sanity-Check Runner Gate | 设计固定 runner id、超时、日志、artifact 和审计边界 | 第一批否 |
| 6 | v1.0 | Internal Preview Closeout | 完成审计、用户手册、操作手册、演示和测试报告 | 只按前置 gate |

## 每一步完成标准

### v0.11 Safe Operations Policy

目标：

- 增加 `lab_root`、`lab_user`、`readonly` 三种内部角色 metadata。
- 增加 safe operation 类型，区分 read-only、review、approved-template dry-run
  和 delete-preview。
- 增加删除 review request/packet，第一版只预览，不删除。
- 测试证明 `lab_root` 也不能绕过 dry-run、路径保护和删除确认。

完成后应提交：

- C++ metadata、validation 和 renderer。
- 单元测试。
- `docs/upgrade/test-report-v0.11.md`。
- 更新 `docs/upgrade/learning-summary-v0.11-safe-operations.md` 或新增
  v0.11 完成学习总结。

### v0.12 Fake Lifecycle

目标：

- 增加单服务器 job lifecycle metadata。
- 支持 requested、reviewed、approved、rejected、queued、running、succeeded、
  failed、cancelled 等状态。
- 只在内存 metadata 中模拟状态变化，不连接服务器、不执行命令、不创建目录。
- 生成 lifecycle review packet，告诉用户现在处于哪个阶段、下一步允许什么。

完成后应提交：

- `single_server_lifecycle` C++ 组件。
- lifecycle 状态转换测试。
- `docs/upgrade/test-report-v0.12.md`。
- `docs/upgrade/learning-summary-v0.12.md`。

### v0.13 Workspace Planner

目标：

- 生成 workspace path、artifact path、log path、run directory preview。
- 校验路径必须在配置的 lab workspace root 之下。
- 拒绝 `..`、绝对路径逃逸、空 workspace root 和危险目录。
- 不创建目录、不删除目录、不移动文件。

完成后应提交：

- `workspace_planner` C++ 组件。
- 路径安全测试。
- `docs/upgrade/test-report-v0.13.md`。
- `docs/upgrade/learning-summary-v0.13.md`。

### v0.14 Approved Template Run Packet

目标：

- 把 approved template、结构化参数、profile、workspace plan 合成 run packet。
- run packet 只表达“将来如何运行”，本阶段不执行。
- 拒绝未知参数、缺失必填参数、未批准 template 和用户自由 command。
- review packet 必须显式写出 `command_executed: false`。

完成后应提交：

- `approved_template_run_packet` C++ 组件。
- 参数验证和 packet rendering 测试。
- `docs/upgrade/test-report-v0.14.md`。
- `docs/upgrade/learning-summary-v0.14.md`。

### v0.15 Internal Sanity-Check Runner Gate

目标：

- 先写清楚真实执行 gate，而不是直接接 SSH、Slurm、PBS 或 CUDA/MPI。
- runner 只能通过固定 allowlisted runner id 选择。
- 不接受用户自由 command string。
- 计划必须包含 timeout、stdout/stderr capture、artifact path、workspace root
  校验和审计事件。
- 第一批实现仍以 metadata、review packet 和测试为主。

完成后应提交：

- sanity-check runner gate 设计文档。
- runner request/review packet metadata。
- 测试证明自由命令、删除请求和凭据读取仍被拒绝。
- `docs/upgrade/test-report-v0.15.md`。
- `docs/upgrade/learning-summary-v0.15.md`。

### v1.0 Internal Preview Closeout

目标：

- 汇总 v0.11 到 v0.15 的安全边界和测试结果。
- 写内部用户手册：如何选 template、填参数、看 review packet、看 lifecycle。
- 写 operator runbook：如何添加 approved template、如何处理失败、如何回滚。
- 写演示脚本：从实验计划到 review packet，再到 lifecycle 和 artifact 预览。
- 只有所有前置 gate 都满足，才把版本标记为 v1.0 internal preview。

完成后应提交：

- `docs/upgrade/v1.0-internal-preview-audit.md`。
- `docs/upgrade/v1.0-internal-user-guide.md`。
- `docs/upgrade/v1.0-internal-operator-runbook.md`。
- `docs/upgrade/demo-script-v1.0-internal-preview.md`。
- `docs/upgrade/test-report-v1.0-internal-preview.md`。
- `docs/upgrade/learning-summary-v1.0-internal-preview.md`。

## 安全边界

当前路线仍不做这些事：

- 不接真实 CUDA/MPI。
- 不接真实 SSH、Slurm、PBS 或远程服务器。
- 不执行用户输入的任意 shell command。
- 不读取真实服务器密码、token、私钥或账号秘密。
- 不把凭据写入仓库。
- 不做真实删除。
- 不移动 trash。
- 不自动应用 Code Agent patch。

## 判定句

只有下面这句话真实成立时，才进入 v1.0 internal preview：

> 实验室成员可以在不写 shell、不碰凭据、不手动找日志的情况下，选择 approved
> template，生成 review packet，走完 lifecycle，看到日志和 artifact 预览；
> 同时系统不会删除代码、环境、共享数据、凭据或其他人的实验结果。

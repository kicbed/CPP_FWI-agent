# v1.0 Internal Preview Operator Runbook

日期：2026-06-23

适用对象：实验室 operator、项目维护者、负责审核模板和演示的人。当前 runbook
只覆盖 internal preview 的非执行工作流，不包含真实服务器运维步骤。

## Operator Responsibilities

operator 负责：

- 维护 approved template metadata。
- 审核用户提交的结构化参数。
- 检查 review packet 和 validation errors。
- 确认 fake lifecycle 状态变化是否合理。
- 拒绝任何自由命令、凭据读取、SSH/Slurm/PBS、真实删除或远程访问请求。
- 在进入真实 backend 前，停止并要求 M11-T1 到 M11-T7 的批准和实现。

operator 不负责，也不能在本版本中执行：

- 手工替系统执行 review packet 中的未来命令。
- 把用户自由文本改写成 shell command。
- 加载真实服务器凭据。
- 创建或清理 workspace。
- 删除文件或移动 trash。

## Add An Approved Template Metadata Entry

当前 approved template 不是生产配置中心，而是 C++ metadata 形态。新增模板时，
保持这些约束：

- `template_id` 必须稳定、可审计。
- `version` 必须显式填写。
- `profile_id` 必须匹配 `SingleServerProfile.profile_id`。
- `entrypoint_label` 只能是开发者维护的固定标签，不能是用户命令。
- `allowed_parameter_names` 只列结构化参数名。
- `expected_artifacts` 只列未来 artifact 类别。
- `max_gpus`、`max_mpi_ranks`、`max_wall_time_minutes` 只表达上限 metadata。

不要新增：

- command string 字段
- credential 字段
- SSH host 字段
- Slurm/PBS submit 字段
- filesystem delete 字段

修改后运行：

```bash
cmake --build build -j2
ctest --test-dir build -R SingleServerBackendTest --output-on-failure
ctest --test-dir build -R ApprovedTemplateRunPacketTest --output-on-failure
```

## Review A User Request

检查顺序：

1. `SingleServerProfile` 是否 metadata-only，且 `runtime_enabled == false`。
2. `SingleServerReviewRequest.dry_run == true`。
3. profile/template/request 的 profile id、template id、version 是否一致。
4. 参数名是否全部来自 template allowlist。
5. required parameters 是否齐全。
6. packet 是否没有渲染 credential reference。
7. packet 是否没有渲染用户自由 command。

出现任意错误时，保持 lifecycle 在 `requested` 或转入 `rejected`，不要进入 approved。

## Review Delete Dry-Run Packet

删除请求第一版只能 preview。operator 必须确认：

- 用户角色允许 `DeleteWorkspaceDryRun`。
- `dry_run == true`。
- `target_path` 在 `workspace_root` 下。
- 目标不是 workspace root 本身。
- 没有 `..` traversal。
- 没有 protected/code/environment/credential/shared dataset 标记。
- 没有 symlink 风险。
- `confirmation_phrase` 精确匹配 `target_path`。
- packet 输出 `deletion_executed: false`、`trash_move_executed: false`、
  `shell_executed: false`。

即使请求来自 `lab_root`，也不能绕过这些检查。

## Review Fake Lifecycle

允许的典型路径：

```text
requested -> reviewed -> approved -> queued -> running -> succeeded
requested -> reviewed -> rejected
approved -> cancelled
running -> failed
```

operator 在每一步检查：

- `allowed_next_states` 是否包含目标状态。
- terminal state 之后没有继续 running。
- preview 仍显示 `server_connected: false`、`command_executed: false`、
  `workspace_created: false`。

`queued` 和 `running` 是 fake lifecycle metadata，不是调度器或进程状态。

## Review Workspace Plan

拒绝这些情况：

- 空 `workspace_root`
- `workspace_root` 为 `/` 或保护目录
- job/run/log/artifact 中包含 `..`
- 相对组件使用绝对路径
- 使用 `.git`、`.ssh`、`secrets`、`credentials`、`repo`、`code`、`env`、
  `venv`、`shared_data` 等保护标签

不要根据 preview 创建目录。preview 只说明未来路径计划。

## Review Sanity Runner Gate

sanity runner gate 是未来真实执行的最小形状评审，不是执行入口。operator 检查：

- runner id 在 allowlist 中。
- fixed entrypoint label 是开发者维护的标签。
- timeout 为正数。
- stdout 和 stderr capture 都有计划。
- artifact path 留在 workspace root 下。
- audit event type 只用于 metadata plan。
- dangerous request flags 都被拒绝。

如果看到 `free_form_command`、delete、credential read、SSH、Slurm、PBS 或 remote
server access 请求，必须拒绝。

## Stop Before Real Execution

看到下面任何需求时立即停止 internal preview 流程：

- 需要运行真实 CUDA/MPI。
- 需要连接服务器或 SSH。
- 需要 Slurm/PBS submit。
- 需要读取真实凭据。
- 需要创建 workspace 或采集真实 artifact。
- 需要删除或清理文件。

进入真实 backend 前必须先完成：

- M11-T1 实验室后端决策包批准。
- M11-T2 authentication and access control。
- M11-T3 workspace creation and cleanup。
- M11-T4 submission, status polling, cancellation。
- M11-T5 log collection and artifact indexing。
- M11-T6 visualization。
- M11-T7 persistent audit logging。

只有这些控制实现并通过评审后，才能讨论改变 runtime backend guard。

## Safety Boundary Audit

The internal preview remains non-executing:

- Users do not provide shell commands.
- `SingleServerProfile.runtime_enabled` remains rejected when true.
- `DeleteReviewPacket` always keeps `deletion_executed`,
  `trash_move_executed`, and `shell_executed` false.
- `SingleServerLifecycleRecord` keeps `server_connected`, `command_executed`,
  and `workspace_created` false.
- `WorkspacePlan` only previews strings and keeps `directories_created`,
  `files_moved`, and `server_connected` false.
- `ApprovedTemplateRunPacket` keeps execution, credentials, server connection,
  workspace creation, directory creation, file movement, and free-form command
  acceptance disabled.
- `SanityRunnerReviewPacket` keeps execution, credentials, deletion, SSH,
  Slurm, PBS, remote access, workspace creation, and artifact capture disabled.

## Explicit Non-Goals

v1.0 internal preview does not include:

- real CUDA/MPI execution
- SSH, Slurm, PBS, local wrapper, or remote execution
- arbitrary shell execution from user text
- credential loading or secret-manager integration
- real deletion, trash moves, workspace cleanup, or file movement
- workspace directory creation
- real stdout/stderr, log, or artifact collection
- production audit persistence
- automatic Code Agent patch application

## Release Decision

Because every required v0.11-v0.15 gate is implemented, documented, and covered
by passing tests, the repository may be marked as `v1.0 internal preview` for
the review-only single-server workflow.

Full real backend work remains gated on M11-T1 through M11-T7: lab backend
approval, credential policy, workspace lifecycle, authentication/access
control, submission/status/cancellation, artifact collection, visualization,
and audit persistence.
